API Overview

OpenStack Reporter API

Version: {{.version}}

REST API for OpenStack resources reporting and management. This API allows you to fetch, cache, and export OpenStack resource information in various formats.

Base URL: http://localhost:8080/api

Authentication

API Token Authentication

Most endpoints require API token authentication. Set the API_TOKEN environment variable to enable authentication.

Note: If API_TOKEN is not set, authentication is disabled for backward compatibility.
Ways to pass the token:
  • Authorization header: Authorization: Bearer <token>
  • X-API-Token header: X-API-Token: <token>
  • Query parameter: ?token=<token> (for EventSource/SSE)
OpenStack Credentials

The API uses OpenStack credentials configured via environment variables:

export OS_PROJECT_DOMAIN_NAME=vhc-pc export OS_USER_DOMAIN_NAME=vhc-pc export OS_USERNAME=your_username export OS_PASSWORD=your_password export OS_AUTH_URL=https://eu3-cloud.virtuozzo.com:5000/v3 export OS_IDENTITY_API_VERSION=3 export OS_AUTH_TYPE=password export OS_INSECURE=true # API Authentication (optional) export API_TOKEN=your-secret-api-token-here
Public Endpoints (no authentication required):
  • GET /api/status
  • GET /api/version
  • GET /api/docs
Protected Endpoints (require API_TOKEN):
  • GET /api/resources
  • GET /api/projects
  • POST /api/refresh
  • POST /api/refresh/progress
  • GET /api/progress
  • GET /api/export/pdf

API Endpoints

GET
/api/resources
Requires Auth

Get all OpenStack resources from cache or fetch from API. Supports filtering via query parameters.

Query Parameters:
  • force (optional) - Force refresh from OpenStack API
  • project (optional) - Filter by project name(s), comma-separated
  • project_id (optional) - Filter by project ID(s), comma-separated
  • type (optional) - Filter by resource type(s), comma-separated
  • status (optional) - Filter by status, comma-separated
Response Example:
{ "projects": [...], "resources": [...], "summary": { "total_projects": 5, "total_servers": 10, "total_volumes": 5, "total_networks": 3, "total_load_balancers": 2, "total_floating_ips": 4, "total_routers": 2, "total_vpn_services": 1, "total_clusters": 1 }, "generated_at": "2025-01-15T10:30:00Z" }
GET
/api/projects
Requires Auth

Get list of all OpenStack projects

Response Example:
{ "projects": [ { "id": "project-id-1", "name": "project-name-1", "description": "Project description", "domain_id": "domain-id", "enabled": true } ], "total": 5, "generated_at": "2025-01-15T10:30:00Z" }
POST
/api/refresh
Requires Auth

Force refresh all resources from OpenStack API

Response Example:
{ "message": "Resources refreshed successfully", "generated_at": "2025-01-15T10:30:00Z", "total_resources": 25 }
POST
/api/refresh/progress
Requires Auth

Force refresh all resources from OpenStack API with progress updates via Server-Sent Events (SSE)

Response:
  • Content-Type: application/json
  • Returns session_id to use with /api/progress endpoint
Response Example:
{ "message": "Refresh started", "session_id": "session_1234567890" }
GET
/api/progress
Requires Auth

Get current refresh progress status via Server-Sent Events (SSE)

Query Parameters:
  • session_id (required) - Session ID from /api/refresh/progress
Response:
  • Content-Type: text/event-stream
  • Streams progress updates as JSON events
GET
/api/export/pdf
Requires Auth

Export current report to PDF format

Response:
  • Content-Type: application/pdf
  • Content-Disposition: attachment; filename=openstack-report.pdf
GET
/api/status
Public

Get current report status and metadata

Response Example:
{ "report_exists": true, "last_check": "2025-01-15T10:30:00Z", "report_age_hours": 0.25, "report_age_human": "15 минут" }
GET
/api/version
Public

Get application version information

Response Example:
{ "version": "v1.0.0", "git_commit": "abc123...", "build_time": "2025-01-15T10:00:00Z", "go_version": "go1.21.0" }
GET
/api/docs
Public

Get this API documentation in JSON format

Supported Resources

  • Projects OpenStack projects/tenants
  • Servers Virtual machines (Nova)
  • Volumes Block storage volumes (Cinder)
  • Networks Network resources with subnets (Neutron)
  • Load Balancers Load balancing services (Octavia)
  • Floating IPs Public IP addresses (Neutron)
  • Routers Network routers (Neutron)
  • VPN Services VPN gateway services (Neutron)
  • Clusters Kubernetes clusters (Magnum)

Resource Filtering

The GET /api/resources endpoint supports filtering via query parameters. Filters can be combined and work as AND conditions.

Available Filters
Project Filter

Filter by project name or ID:

# By project name GET /api/resources?project=infra,production # By project ID GET /api/resources?project_id=123,456
Type Filter

Filter by resource type (comma-separated):

GET /api/resources?type=server,volume,network

Available types: server, volume, network, load_balancer, floating_ip, router, vpn_service, cluster

Status Filter

Filter by status (comma-separated):

GET /api/resources?status=active,available
Combined Filters

Combine multiple filters:

# Servers and volumes from infra project with active status GET /api/resources?project=infra&type=server,volume&status=active
Note: The summary is automatically recalculated for filtered results.

Usage Examples

cURL Examples:
Get all resources (with authentication):
curl -H "Authorization: Bearer YOUR_TOKEN" \ "http://localhost:8080/api/resources"
Get filtered resources:
curl -H "Authorization: Bearer YOUR_TOKEN" \ "http://localhost:8080/api/resources?project=infra&type=server,volume"
Get list of projects:
curl -H "Authorization: Bearer YOUR_TOKEN" \ "http://localhost:8080/api/projects"
Refresh resources:
curl -X POST \ -H "Authorization: Bearer YOUR_TOKEN" \ "http://localhost:8080/api/refresh"
Refresh with progress (SSE):
# Start refresh curl -X POST \ -H "Authorization: Bearer YOUR_TOKEN" \ "http://localhost:8080/api/refresh/progress" # Get progress (use session_id from above) curl -H "Authorization: Bearer YOUR_TOKEN" \ "http://localhost:8080/api/progress?session_id=session_1234567890"
Download PDF report:
curl -H "Authorization: Bearer YOUR_TOKEN" \ "http://localhost:8080/api/export/pdf" -o report.pdf
Check API status (public endpoint):
curl "http://localhost:8080/api/status"
Get version (public endpoint):
curl "http://localhost:8080/api/version"
